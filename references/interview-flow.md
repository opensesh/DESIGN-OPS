# Interview Flow Reference

Detailed question orchestration for the `/install-skills` onboarding.

## Design Principles

1. **Light and fast** — 2 minutes max, 3 core questions
2. **No wrong answers** — everything works at any integration level
3. **Show, don't tell** — demonstrate value immediately
4. **Graceful defaults** — if user skips, pick sensible options

---

## Question Details

### Question 1: Tools Used

**Goal:** Build context about their work environment

**Ask:**
```
What tools do you use for work?
(Just list them — Calendar, Notion, Slack, whatever you use)
```

**Input type:** Freeform text

**Parsing logic:**
- Split by commas, newlines, or "and"
- Normalize tool names (e.g., "google cal" → "Google Calendar")
- Store as clean list

**Common tool mappings:**

| User Says | Normalized |
|-----------|------------|
| cal, gcal, google cal | Google Calendar |
| notion | Notion |
| slack | Slack |
| github, gh | GitHub |
| linear | Linear |
| jira | Jira |
| asana | Asana |
| figma | Figma |
| gmail, email | Gmail |
| outlook | Outlook |
| supabase | Supabase |

**Why this matters:**
- Helps Claude understand context in conversations
- Informs which MCPs might be useful
- Personalizes command output

### Question 2: Workflow Rhythm

**Goal:** Understand when they want Claude's help

**Present as checkboxes using AskUserQuestion:**

```
How do you want help organized?
(Select what sounds useful)
```

**Options:**

**Daily:**
- [ ] Morning brief — start the day with priorities
- [ ] Meeting prep — get ready for upcoming meetings

**Weekly:**
- [ ] Weekly recap — reflect on the week, plan the next

**As-needed:**
- [ ] Decision support — challenge assumptions, pre-mortems
- [ ] Research — analyze websites, competitors
- [ ] Content — social posts, case studies

**Mapping:**

| Selection | Config Key | Commands |
|-----------|------------|----------|
| Morning brief | `morning_brief` | `/daily-brief` |
| Meeting prep | `meeting_prep` | `/meeting-brief`, `/meeting-recap` |
| Weekly recap | `weekly_recap` | `/weekly-recap` |
| Decision support | `decision_support` | `/devils-advocate`, `/pre-mortem` |
| Research | `research` | `/site-analysis` |
| Content | `content` | `/social-post`, `/case-study` |

**Default if nothing selected:** Enable all (with note that they can customize later)

---

### Question 3: MCP Status Check

**Goal:** Show what's connected and offer quick setup

**Logic:**

1. **Detect connected MCPs** by checking available MCP tools
2. **Categorize** into connected vs. recommended-not-connected
3. **Present status** with what each enables

**Detection approach:**
- Check for MCP tool availability in current session
- Common MCPs to look for: GitHub, Supabase, Google Calendar, Notion, Firecrawl, Gmail

**Display format:**

```
Checking your connected tools...

✅ GitHub — ready
   What it enables: code search, PR reviews, issue tracking

✅ Supabase — ready
   What it enables: database queries, user management

⚠️ Google Calendar — not connected
   Would enable: auto-fetch events for daily briefs, meeting prep

⚠️ Notion — not connected
   Would enable: task lists, project tracking, note search
```

**Follow-up:**
```
Would you like to set up any missing tools now?
```

Options:
- Set up [specific tool] — link to MCP setup guide
- Skip for now — commands work without them, just with less automation

**Important:** Never make this a blocker. Everything should work at any integration level.

## Adaptive Behaviors

### First-Time User
- Full interview flow
- Extra context in explanations
- Link to `/skills-help` for later

### Returning User (Config Exists)
- Show current config summary
- Ask what to change:
  - "Add more tools"
  - "Change workflow preferences"
  - "Connect new MCP"
  - "Start fresh"
- Only re-ask relevant questions

### Power User (Wants to Skip)
- If user says "just set it up" or similar
- Enable all commands
- Use detected MCPs
- Note they can customize with `/customize`

## Config Generation

After interview, generate YAML:

```yaml
# Company Skills Configuration
# Generated by /install-skills

version: 1.0
created: 2026-04-03T10:30:00Z

tools_used:
  - Google Calendar
  - Notion
  - Slack
  - GitHub

mcp_connected:
  - name: github
    capabilities: [code_search, pull_requests, issues]
  - name: supabase
    capabilities: [database, auth, storage]

workflows:
  daily:
    - morning_brief
    - meeting_prep
  weekly:
    - weekly_recap
  as_needed:
    - devils_advocate
    - pre_mortem
    - site_analysis
```

**Rules:**
- Only include sections that have values
- No `enabled: false` flags — omit instead
- Include capabilities for each MCP (helps commands know what's available)

## Edge Cases

### User provides no tools
- Ask: "No worries! Do you use any calendar or task app?"
- If still nothing: proceed without, commands work standalone

### User selects nothing for workflows
- Enable all by default
- Say: "I've enabled everything — run `/customize` later to narrow down."

### MCP detection fails
- Skip the MCP question
- Note: "I couldn't detect your connected tools. Run `/add-tool` later to connect them."

### Config write fails
- Show the YAML content
- Provide manual creation instructions
- Don't block — let them use commands anyway
